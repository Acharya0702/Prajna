Chapter 1: Introduction

The SQL Language

SQL is a tool for organizing, managing, and retrieving data stored by a computer database. The name "SQL" is an abbreviation for Structured Query Language. For historical reasons, SQL is usually pronounced "sequel," but the alternate pronunciation "S.Q.L." is also used. As the name implies, SQL is a computer language that you use to interact with a database. SQL works with one specific type of database, called a relational database.

Figure 1-1 shows how SQL works. The computer system in the figure has a database that stores important information. If the computer system is in a business, the database might store inventory, production, sales, or payroll data. On a personal computer, the database might store data about the checks you have written, lists of people and their phone numbers, or data extracted from a larger computer system. The computer program that controls the database is called a database management system, or DBMS.

When you need to retrieve data from a database, you use the SQL language to make the request. The DBMS processes the SQL request, retrieves the requested data, and returns it to you. This process of requesting data from a database and receiving back the results is called a database query—hence the name Structured Query Language.

SQL is a comprehensive language for controlling and interacting with a database management system, including:

  • Data definition - Define the structure and organization of stored data
  • Data retrieval - Retrieve stored data from the database
  • Data manipulation - Update, add, and remove data
  • Access control - Restrict user access to protect data
  • Data sharing - Coordinate data sharing by concurrent users
  • Data integrity - Define integrity constraints to prevent corruption

Chapter 2: A Quick Tour of SQL

A Simple Database

The examples in this tutorial are based on a simple relational database for a small distribution company. The database stores:

  • Customers who buy the company's products
  • Orders placed by those customers
  • Salespeople who sell products to customers
  • Sales offices where salespeople work

Retrieving Data (SELECT)

The SELECT statement retrieves data from the database. Here's an example listing sales offices:

Chapter 3: SQL In Perspective

A Brief History of SQL

SQL's history is intertwined with the development of relational databases. In June 1970, Dr. E.F. "Ted" Codd published "A Relational Model of Data for Large Shared Data Banks," outlining a mathematical theory of how data could be stored and manipulated using a tabular structure.

DateEvent
1970Codd defines relational database model
1974IBM's System/R project begins
1979Oracle introduces first commercial RDBMS
1986ANSI SQL1 standard ratified
1992ANSI SQL2 standard ratified

SQL has emerged as the standard database language for using relational databases, offering vendor independence, portability across systems, and strong industry support from IBM and Microsoft.

Chapter 4: Relational Databases

Tables

The organizing principle in a relational database is the table, a rectangular, row/column arrangement of data values. Each table in a database has a unique table name that identifies its contents.

Primary Keys and Foreign Keys

Primary Key: A column or combination of columns that uniquely identifies each row in a table.

Foreign Key: A column in one table whose values match the primary key in another table, creating relationships between tables.

Chapter 6: Simple Queries

The SELECT Statement

The SELECT statement consists of six clauses:

  • SELECT - Lists data items to retrieve
  • FROM - Lists tables containing the data
  • WHERE - Specifies rows to include (optional)
  • GROUP BY - Groups rows for summary (optional)
  • HAVING - Filters groups (optional)
  • ORDER BY - Sorts query results (optional)

WHERE Clause Examples

-- Comparison test
SELECT NAME, SALES FROM SALESREPS WHERE SALES > QUOTA

-- Range test (BETWEEN)
SELECT NAME, SALES FROM SALESREPS WHERE SALES BETWEEN 100000 AND 500000

-- Set membership (IN)
SELECT CITY FROM OFFICES WHERE REGION IN ('Eastern', 'Western')

-- Pattern matching (LIKE)
SELECT NAME FROM CUSTOMERS WHERE COMPANY LIKE 'A%'

-- NULL test
SELECT NAME FROM SALESREPS WHERE MANAGER IS NULL

ORDER BY Clause

-- Sort by region ascending, then sales descending
SELECT CITY, REGION, SALES FROM OFFICES 
ORDER BY REGION ASC, SALES DESC

Chapter 7: Multi-Table Queries (Joins)

Joining Tables

Multi-table queries combine data from two or more tables based on common values. The link between tables is created through foreign keys.

-- List orders with customer names
SELECT ORDER_NUM, AMOUNT, COMPANY, CREDIT_LIMIT 
FROM ORDERS, CUSTOMERS
WHERE CUST = CUST_NUM

Types of Joins

  • INNER JOIN - Returns rows with matching values in both tables
  • LEFT JOIN - Returns all rows from left table, matched rows from right
  • RIGHT JOIN - Returns all rows from right table, matched rows from left
  • FULL OUTER JOIN - Returns all rows when there's a match in either table
