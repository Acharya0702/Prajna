Chapter 1: What is Android?

Key Skills & Concepts

  • History of embedded device programming
  • Explanation of Open Handset Alliance
  • First look at the Android home screen

It can be said that, for a while, traditional desktop application developers have been spoiled. This is not to say that traditional desktop application development is easier than other forms of development. However, as traditional desktop application developers, we have had the ability to create almost any kind of application we can imagine.

Brief History of Embedded Device Programming

For a long time, cell phone developers comprised a small sect of a slightly larger group of developers known as embedded device developers. Seen as a less "glamorous" sibling to desktop—and later web—development, embedded device development typically got the proverbial short end of the stick as far as hardware and operating system features.

Open Handset Alliance and Android

This barrier to application development began to crumble in November of 2007 when Google, under the Open Handset Alliance, released Android. The Open Handset Alliance is a group of hardware and software developers, including Google, NTT DoCoMo, Sprint Nextel, and HTC, whose goal is to create a more open cell phone environment.

Google, in promoting the new Android operating system, even went as far as to create a $10 million contest looking for new and exciting Android applications.

Introduction to Android

Android, as a system, is a Java-based operating system that runs on the Linux 2.6 kernel. The system is very lightweight and full featured. Android applications are developed using Java and can be ported rather easily to the new platform.

Other features of Android include an accelerated 3-D graphics engine (based on hardware support), database support powered by SQLite, and an integrated web browser.

Chapter 2: Downloading and Installing Eclipse

Key Skills & Concepts

  • Selecting a development environment
  • Downloading Eclipse
  • Installing and configuring Eclipse

Android applications are developed in Java. Android itself is not a language, but rather an environment within which to run applications. As such, you can theoretically use any distribution or integrated development environment (IDE) you have at your disposal to begin your development.

Why Eclipse?

Eclipse is the recommended IDE for Android applications for several reasons:

  • It is one of the most fully featured, free Java IDEs available
  • The Open Handset Alliance has released an Android plugin for Eclipse
  • Eclipse is available for Mac, Windows, and Linux

Downloading the Android SDK

The Android SDK is easily accessible from the http://code.google.com/android page. From the development home page, click the Download the SDK link under Getting Started.

Configuring the Android Plugin for Eclipse

After installing the Android plugin for Eclipse, restart Eclipse. The Android plugin for Eclipse is configured from the Preferences window of Eclipse.

What Are Intents?

An Intent is Android's method for relaying certain information from one Activity to another. An Intent, in simpler terms, expresses to Android your intent to do something. You can think of an Intent as a message passed between Activities.

Activity Action IntentMessage
DIAL_ACTIONOpen the Dial Activity and dial the specified number
CALL_ACTIONPlace a call to supplied location
WEB_SEARCH_ACTIONOpen Google Search, or another web page if specified
 
Intent callIntent = new Intent(Intent.CALL_ACTION, Uri.parse("tel:5551212"));
callIntent.setLaunchFlags(Intent.NEW_TASK_LAUNCH);
startActivity(callIntent);
